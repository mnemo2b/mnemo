# Research: Context Switching

## Question

What's the actual cost of context switching — cognitive, temporal, qualitative? Does it apply equally across task types?

## Core findings

- **Monsell (2003)** — classic task switching paradigms show measurable switch costs (often 100-200ms per switch, larger for complex tasks)
- **Mark et al. (2008, UC Irvine)** — office workers spend ~3 minutes on a task before switching; full recovery of attention takes 23 minutes
- **Leroy (2009)** — attention residue persists across tasks; incomplete tasks degrade performance on the next one more than completed ones
- **Eriksen & Eriksen (1974)** — flanker task, attention narrows and broadens; switching contexts requires re-tuning

## What makes switching costly

- Task-set reconfiguration (mental model loading)
- Proactive interference (previous task's content intrudes)
- Working memory contention
- Emotional residue (unresolved task creates low-grade anxiety)

## What makes switching cheaper

- Similar tasks (switching between two emails vs email to code)
- Well-practiced domains (expert multitaskers in limited cases)
- External scaffolding (notes, tools that rebuild context quickly)
- Completion before switching (Zeigarnik effect — unfinished tasks consume attention)

## Rotating myths

- **"Context switching is free for good multitaskers"** — self-perception diverges from performance; multitaskers rate themselves high, perform poorly
- **"Media multitasking trains your brain"** — research suggests opposite: degrades working memory filtering
- **"You only need 20 minutes of focus to be productive"** — some tasks, sure; complex cognitive work needs more
- **"Context switching is inevitable, embrace it"** — inevitable ≠ costless

## Mitigations

- **Batch similar work** — email blocks, meeting blocks, code blocks
- **Explicit pause rituals** — write down what you were doing before interruption
- **Environmental scaffolding** — IDE state, notebook, whiteboard retain context
- **Calendar defense** — interruption-free blocks as appointments with yourself
- **Single-tasking tools** — hiding other apps, disabling notifications

## Open questions

- Does chronic context switching degrade baseline cognitive performance or just momentary?
- How do async communication norms (Slack, Discord) compare to synchronous ones?
- Can better tools (AI assistants, codified state) reduce switching costs meaningfully?
