# Projects

Things I'm building. Each directory is a project — its context, decisions, and implementation notes live together.

## What goes here

- Project-specific decisions tied to this codebase, data shape, or stack
- Architecture, implementation choices, roadmap, active questions
- Anything that would be lost context if it moved out of the project

## What doesn't

- Reusable principles and patterns — those go in `notes/` topics, even if the project is what surfaced them
- Raw source material (papers, articles) — those go in `research/`

When an insight starts in a project but generalizes, write it in the relevant topic and keep only the project-specific application here.

## Conventions

- one directory per project (kebab-case)
- a `README.md` anchors each project with shape, status, and links
- other files are scoped to concerns (`architecture.md`, `roadmap.md`, etc.)
